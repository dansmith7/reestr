## Один обязательный шаг, который я не могу сделать сам

Vercel даёт мне доступ к деплою кода, но не к дашборду проекта — подключение базы данных и изменение переменных окружения возможны только руками через сайт vercel.com. Без этого шага сайт откроется, но при обновлении статуса будет ошибка "не найдена база данных".

**Подключить Postgres (2 минуты, один раз):**

1. Зайдите на vercel.com → откройте проект **reestr-status**.
2. Вкладка **Storage** → **Create Database** → выберите **Postgres** (Neon, бесплатный тариф хватает с большим запасом).
3. На шаге подключения выберите **Connect to Project** → `reestr-status` → окружение **Production**.
4. Vercel сам добавит переменную `POSTGRES_URL` в настройки проекта.
5. Вкладка **Deployments** → у последнего деплоя нажмите `...` → **Redeploy** (нужно, чтобы приложение подхватило новую переменную).

После этого при первом открытии сайта база автоматически создастся и заполнится вашими текущими 6 проектами и 15 этапами — ничего вручную вносить не нужно.

## Как сменить пароль

1. Проект → **Settings** → **Environment Variables**.
2. Добавьте `APP_PASSWORD` со своим значением (Production).
3. **Deployments** → **Redeploy**.

## Что умеет

То же самое, что в локальной версии: обновление статуса с комментарием и сроком, история изменений по каждому проекту, флаг «застрял» (нет изменений дольше 2 недель), сравнение текущего состояния с любой прошедшей датой, управление списком проектов и этапов через интерфейс.

## Структура

```
registry-app-vercel/
  middleware.js         — защита паролем (Basic Auth)
  lib/db.js              — подключение к Postgres, схема, начальные данные
  pages/api/             — backend (data, status, projects, stages, history, compare)
  public/index.html      — интерфейс (тот же дизайн)
```
